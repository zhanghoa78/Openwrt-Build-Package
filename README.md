LuCI App for TCPDump (JavaScript)luci-app-tcpdump-js 是一款为 OpenWrt / ImmortalWrt 路由器设计的现代化 LuCI (Web) 插件。它提供了一个简洁的图形界面，用于远程控制路由器上的 tcpdump 命令行工具，帮助您轻松抓取网络数据包。本项目使用现代 LuCI JS 视图和后端 Lua RPC 实现，确保了插件的轻量和平台无关性（_all.ipk）。界面预览[一个简洁界面的插图]状态显示： 清晰显示抓包进程是 "RUNNING" (运行中) 还是 "STOPPED" (已停止)。参数配置： 提供下拉菜单和输入框，用于配置抓包参数。智能控制： 按钮状态（启用/禁用）会根据抓包状态自动更新。主要功能远程启停： 通过 LuCI 界面安全地启动和停止 tcpdump 进程。状态轮询： 自动（每3秒）刷新抓包进程的状态 (PID)，实时反馈在界面上。接口选择： 自动检测并罗列路由器上所有可用的网络接口（例如 br-lan, eth0, wlan0）供您选择。BPF 过滤器： 支持输入标准的 BPF 过滤语法（例如 port 80 或 host 1.1.1.1）。大小限制： (新功能) 可以设置一个抓包文件的大小限制（单位：MB）。当抓包文件 (.pcap) 达到该大小时，tcpdump 进程会自动停止，防止占满 /tmp 目录。依赖项为了使本插件正常工作，您的路由器上必须安装以下软件包：luci-base (LuCI 基础框架)luci-lib-nixio (LuCI I/O 库)luci-mod-rpc (LuCI RPC 接口库)tcpdump (核心抓包工具)当您使用 GitHub Actions 工作流编译本插件时，tcpdump 和 luci-* 依赖项的 .ipk 包会与本插件一起被编译和打包。安装方法获取 IPK： 从本项目的 GitHub Actions 或 Releases 页面下载编译好的 luci-app-tcpdump-js_..._all.ipk 文件。同时下载与您路由器架构相符的 tcpdump_..._ARCH.ipk 文件（如果您的固件中尚未包含它）。上传 IPK： 将下载的 .ipk 文件上传到路由器的 /tmp 目录（例如使用 scp 或 WinSCP）。安装插件： 通过 SSH 登录路由器，然后执行以下命令：# (如果需要) 先安装依赖项
opkg install /tmp/tcpdump_..._ARCH.ipk

# 安装本插件
opkg install /tmp/luci-app-tcpdump-js_..._all.ipk
清除缓存： 安装完成后，清除 LuCI 菜单缓存：rm /tmp/luci-indexcache
使用指南安装成功后，刷新 LuCI 界面。导航至 网络 (Network) > TCPDump。在“设置”卡片中：Interface (接口)： 选择您想监听的网络接口。Filter (过滤器)： (可选) 输入您的 BPF 过滤规则。Size Limit (MB) (大小限制)： (可选) 输入一个数字（例如 10），限制抓包文件最大为 10MB。点击绿色的 [Start Capture] (开始抓包) 按钮。状态将变为 RUNNING。抓包完成后（手动点击停止或达到大小限制），状态将变回 STOPPED。使用 scp 或 WinSCP 等工具，从路由器的 /tmp/tcpdump.pcap 路径下载抓包文件。使用 Wireshark 或其他抓包分析软件打开 .pcap 文件进行分析。注意事项安全： 本插件的 Lua 后端对 BPF 过滤器的输入仅做了基础的转义，请勿输入恶意或格式错误的字符串。存储： 抓包文件保存在 /tmp 目录，这是路由器的内存（RAM）空间。抓包文件不应设置得过大，以免耗尽路由器内存导致重启。建议始终设置“大小限制”。下载： 本插件不提供“在线查看”或“点击下载”功能，您必须通过 scp 或 WinSCP 手动从 /tmp/tcpdump.pcap 下载文件。
